---
title: "Update Visibility"
api: "PATCH /packages/{package_id}/visibility"
description: "Change the visibility of a package (private / internal / public)"
---

Update the visibility of a package without redeploying. The `visibility` field
in `mecapy.yml` sets the initial value at deploy time; use this endpoint to
change it afterwards — for example, to publish a package after internal
validation.

See [Visibility](/security/visibility) for the transition rules.

## Path Parameters

<ParamField path="package_id" type="string" required>
  Unique identifier of the package (UUID)
</ParamField>

## Request Body

<ParamField body="visibility" type="string" required>
  One of `private`, `internal`, `public`.
</ParamField>

<Note>
- Only the **owner** can toggle between `private` and `internal` (on org packages)
  or between `private` and `public` (on personal packages).
- Transition **to `public`** for an organization-owned package requires an
  **Admin of the organization**.
- **`internal`** requires the package to be attached to an organization.
</Note>

<RequestExample>

```bash Publish
curl -X PATCH https://api.mecapy.com/packages/17ad.../visibility \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"visibility": "public"}'
```

```bash Make internal
curl -X PATCH https://api.mecapy.com/packages/17ad.../visibility \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"visibility": "internal"}'
```

</RequestExample>

<ResponseExample>

```json 200 Success
{
  "id": "17ad...",
  "name": "e25-030-1",
  "version": "1.0.0",
  "visibility": "public",
  "organization_id": "8f3c...",
  "functions": [...]
}
```

```json 403 Non-admin publishing org package
{"detail": "PermissionDenied: Only an organization admin can publish this package"}
```

```json 403 Not the owner
{"detail": "PermissionDenied: Only the package owner can change visibility"}
```

```json 403 Internal on personal package
{"detail": "PermissionDenied: Cannot set 'internal' on a personal package (no organization)"}
```

</ResponseExample>
