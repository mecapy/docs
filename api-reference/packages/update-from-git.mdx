---
title: "Update Package from Git"
api: "POST /packages/{package_id}/versions/from-git"
description: "Deploy a new version of an existing package from a Git repository"
---

Deploy a new version of an existing package from a Git repository. Unlike
[`POST /packages/from-git`](/api-reference/packages/deploy-from-git) which creates
a brand new package, this endpoint adds a new version to an existing one while
preserving full history: previous versions remain accessible, only the `is_latest`
pointer moves to the new version.

<Note>
  Only the package owner can update a package. The manifest must declare the
  same package name and a version strictly greater than the current one (SemVer).
</Note>

## Path Parameters

<ParamField path="package_id" type="string" required>
  Unique identifier of the existing package
</ParamField>

## Request Body

<ParamField body="git_url" type="string" required>
  Git repository URL (GitHub, GitLab, Bitbucket)
</ParamField>

<ParamField body="git_ref" type="string" default="main">
  Git reference (branch, tag, or commit SHA) pointing to the new version
</ParamField>

<ParamField body="git_token" type="string">
  Authentication token for private repositories
</ParamField>

<ParamField body="deploy_strategy" type="string" default="all">
  Deployment strategy: `all` or `selective`
</ParamField>

<ParamField body="functions" type="array">
  Function names to deploy (required if strategy is `selective`)
</ParamField>

## Validation Rules

| Condition | Status |
| --- | --- |
| Caller is the package owner | `403` otherwise |
| Package exists | `404` otherwise |
| Manifest name matches the existing package name | `409` otherwise |
| Manifest version is strictly greater than current version (SemVer) | `409` otherwise |
| Manifest version does not already exist for this package | `409` otherwise |
| Manifest version is not older than the latest SemVer Git tag | `409` otherwise |

<RequestExample>

```bash cURL
curl -X POST https://api.mecapy.com/packages/{package_id}/versions/from-git \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "git_url": "https://github.com/mecapy/boulonnerie.git",
    "git_ref": "v1.3.0",
    "deploy_strategy": "all"
  }'
```

```python Python
response = requests.post(
    f"https://api.mecapy.com/packages/{package_id}/versions/from-git",
    headers={"Authorization": f"Bearer {token}"},
    json={
        "git_url": "https://github.com/mecapy/boulonnerie.git",
        "git_ref": "v1.3.0",
        "deploy_strategy": "all",
    },
)
```

</RequestExample>


```json 409 — version not incremented

  "detail": "New version 1.2.0 must be strictly greater than current version 1.2.0 for package boulonnerie. Please increment the version in mecapy.yml."

```

```json 403 — not the owner
{
  "detail": "Only the package owner can update this package"
}
```

</ResponseExample>
